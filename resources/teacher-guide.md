# 教師指引

## 課程定位

這份課綱適合作為高中社團、彈性學習、微課程或自主學習框架。它不假設學生要成為工程師，但要求學生練習工程式的問題拆解、測試與紀錄。

## 建議授課方式

- 每週 1 次 90 分鐘課堂，加上 2 到 4 小時自主練習。
- 課堂只講最小必要觀念，保留時間讓學生實作。
- 每週挑 2 到 3 位學生展示卡關過程，不只展示成功結果。
- 第 11 週以後鼓勵學生把資料換成自己的題目，例如社團出席、運動紀錄、讀書時間、校園問卷。

## 分層教學

- 基礎學生：完成必做題即可，重點放在能執行與能解釋。
- 進階學生：做挑戰題，加入檔案、圖表、API 或 GUI。
- 快速完成者：協助同學 code review，但只能提示，不直接代寫。

## 不建議一開始要求

- 複雜演算法競賽題。
- 大量套件與框架。
- 只看影片不實作。
- 沒有 README 的專題。
