# 教師指引

## 課程定位

這份課綱適合作為高中社團、彈性學習、微課程或自主學習框架。它不假設學生要成為工程師，但要求學生練習工程式的問題拆解、測試與紀錄。若學生完全零基礎，建議先花 1 到 2 週補 `resources/computer-basics.md` 的檔案、路徑、終端機與錯誤訊息概念。

## 建議授課方式

- 每週 1 次 90 分鐘課堂，加上 2 到 4 小時自主練習。
- 第 0 週不要急著進語法，先確認每位學生都能找到檔案、打開終端機、進入正確資料夾並執行 `hello.py`。
- 課堂只講最小必要觀念，保留時間讓學生實作。
- 每週挑 2 到 3 位學生展示卡關過程，不只展示成功結果。
- 第 11 週以後鼓勵學生把資料換成自己的題目，例如社團出席、運動紀錄、讀書時間、校園問卷。
- 每次指定影片最多 1 到 3 支，影片後要接一個短任務，不把觀看本身當成學會。

## 分層教學

- 基礎學生：完成必做題即可，重點放在能執行與能解釋。
- 進階學生：做挑戰題，加入檔案、圖表、API 或 GUI。
- 快速完成者：協助同學 code review，但只能提示，不直接代寫。

## 不建議一開始要求

- 複雜演算法競賽題。
- 大量套件與框架。
- 只看影片不實作。
- 沒有 README 的專題。
