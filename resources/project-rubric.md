# 期末專題評量規準

| 項目 | 比重 | 說明 |
|---|---:|---|
| 可執行性 | 25% | 依 README 能順利安裝與執行 |
| 功能完整度 | 25% | 核心功能可用，錯誤輸入有基本處理 |
| 程式結構 | 20% | 命名清楚，有函式拆解，避免大量重複 |
| 資料處理 | 15% | 能讀寫檔案、CSV、API 或整理資料 |
| 表達與反思 | 15% | Demo 清楚，能說明卡關與下一版規劃 |

## 評語建議

回饋請具體描述行為，例如：

- 「第 32 行可以拆成函式，因為第 58 行也有類似邏輯。」
- 「README 有執行指令，但缺少範例輸出。」
- 「圖表有標題，但 y 軸單位不清楚。」
