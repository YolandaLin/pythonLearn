# 電腦基礎補強

這份文件給完全零基礎學生使用。正式寫 Python 前，先確認自己知道電腦檔案、資料夾、終端機與網路的基本概念。很多初學者不是卡在 Python，而是卡在「程式檔放在哪裡」或「終端機現在在哪個資料夾」。

## 1. 檔案與資料夾

### 必懂概念

- 檔案：電腦裡的一份資料，例如 `hello.py`、`scores.csv`、`README.md`。
- 資料夾：用來分類檔案的位置。
- 副檔名：檔名最後的 `.py`、`.txt`、`.csv`，用來表示檔案類型。
- 路徑：檔案或資料夾的位置，例如 `C:\Users\you\Desktop\hello.py`。
- 相對路徑：從目前資料夾出發的位置，例如 `data\scores.csv`。

### 學生要會做

- 建立資料夾。
- 顯示副檔名。
- 分辨 `hello.py` 和 `hello.py.txt`。
- 知道下載的檔案通常在哪裡。
- 把課程檔案集中放在同一個資料夾，不要散在桌面與下載資料夾。

## 2. 終端機與目前位置

終端機是用文字指令操作電腦的工具。學 Python 不需要一開始會很多指令，但至少要知道「目前位置」很重要。

### Windows 常用指令

```powershell
pwd                 # 顯示目前位置
dir                 # 列出目前資料夾內容
cd Desktop          # 進入 Desktop 資料夾
cd ..               # 回上一層
python --version    # 確認 Python 是否安裝成功
python hello.py     # 執行 Python 程式
```

### 常見錯誤

- 終端機不在程式所在資料夾，所以 `python hello.py` 找不到檔案。
- 檔案其實叫 `hello.py.txt`，不是 `hello.py`。
- 把指令打進 Python 互動模式 `>>>` 裡，而不是打在終端機。

## 3. 程式怎麼執行

Python 程式的流程可以先用這個模型理解：

1. 你寫一份 `.py` 原始碼。
2. Python 直譯器讀取這份檔案。
3. Python 從上到下一行一行執行。
4. 遇到錯誤時，終端機會顯示 traceback。
5. traceback 會告訴你檔名、行號與錯誤類型。

### 初學者讀錯誤訊息的順序

1. 先找自己的檔名。
2. 看行號。
3. 看錯誤類型，例如 `SyntaxError`、`NameError`、`ValueError`。
4. 回到程式碼，只改最可能出錯的一小段。

## 4. 網路基礎

第 13 週會碰到 API。在那之前，先知道這些詞即可：

- 網址 URL：網頁或 API 的地址。
- domain：像 `github.com` 這種給人看的名稱。
- DNS：把 domain 轉成電腦看得懂的位置。
- HTTP：瀏覽器和網站溝通的規則。
- request：你的電腦送出的請求。
- response：伺服器回傳的結果。
- JSON：常見的資料格式，Python 讀進來後通常會變成 dict 或 list。

## 5. 安全底線

- 不要把密碼寫進程式碼。
- 不要把 API key、token、帳號密碼 commit 到 GitHub。
- 看到要求輸入密碼或 token 的網站，要確認網址正確。
- 不要隨便執行陌生人提供的指令，尤其是會刪除檔案或下載程式的指令。
