# 第 9 週自我檢討：錯誤處理與除錯

## 交作業前檢查

- [ ] 能從 traceback 找到檔名與行號
- [ ] 知道不要用空白 except 吃掉所有錯誤
- [ ] 能把問題縮小到最小範例

## 常見錯誤

- 只讓程式在一組範例輸入下成功，沒有測邊界情況。
- 變數名稱太籠統，例如 `a`、`b`、`x`，導致一週後看不懂。
- 沒有在 README 寫如何執行程式。
- 把錯誤訊息只看最後一行，沒有回頭看檔名與行號。

## 修正方向

1. 至少準備三組測試資料：正常、邊界、錯誤輸入。
2. 把重複出現的邏輯整理成函式。
3. 用自己的話寫下本週最重要的觀念。
4. 若程式有使用外部檔案，確認檔名與相對路徑正確。
