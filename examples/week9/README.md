## 常見錯誤

### 1. 只看錯誤訊息最後一行

最後一行只告訴你錯誤類型。請往上找自己的檔名與行號，那才是最重要的位置。

### 2. except 寫太大

不建議：

```python
try:
    number = int(input("數字："))
except:
    print("錯了")
```

比較好：

```python
try:
    number = int(input("數字："))
except ValueError:
    print("請輸入整數")
```

### 3. try 包太多程式

try 裡面只放可能出錯的那幾行。包太大會讓你不知道真正錯在哪裡。

### 4. 用 print 除錯後忘記移除

除錯時可以印出變數，但交作業前要確認不必要的 debug 訊息已移除。

## 修正方向

1. 先複製完整 traceback 到筆記。
2. 找檔名與行號。
3. 用自己的話寫錯誤原因。
4. 只改一小段，再重新執行。
5. 把修法記錄下來，之後會很有用。

## 延伸練習
