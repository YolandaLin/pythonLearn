# 第 0 週自我檢討：電腦基礎、環境準備與學習方法

## 交作業前檢查

- [ ] 能顯示檔案副檔名，分辨 `hello.py` 與 `hello.py.txt`。
- [ ] 能在終端機執行 `python --version`。
- [ ] 能用 `cd` 進入 `hello.py` 所在資料夾。
- [ ] 能執行 `python hello.py`。
- [ ] 能從錯誤訊息找到檔名與行號。
- [ ] 知道如何用 Git commit 一次變更。

## 常見錯誤

- 把 `python hello.py` 打在 Python 互動模式 `>>>` 裡，而不是終端機。
- 終端機目前位置不在 `hello.py` 所在資料夾。
- 檔案實際名稱是 `hello.py.txt`。
- 安裝了 Python，但 VS Code 選到錯的 interpreter。
- README 只寫「成功了」，沒有寫執行指令與輸出結果。
- 變數名稱太籠統，例如 `a`、`b`、`x`，導致一週後看不懂。

## 修正方向

1. 先用 `pwd` 或看終端機提示確認目前位置。
2. 用 `dir` 檢查目前資料夾是否真的有 `hello.py`。
3. 從錯誤訊息找自己的檔名與行號，不要只看最後一行。
4. README 至少要包含版本號、執行指令、執行結果。
5. 如果同學照你的 README 跑不起來，代表 README 還不夠完整。
