# 第 16 週自我檢討：專題完成與成果發表

## 交作業前檢查

- [ ] README 能讓同學照著跑起作品。
- [ ] 程式有至少三組測試資料或操作截圖。
- [ ] final_report.md 有寫卡關、修正與下一版。
- [ ] Demo 不超過 5 分鐘。
- [ ] feedback.md 有兩位同學的具體回饋。

## 常見錯誤

### 1. 最後一週還在加大功能

第 16 週應該收尾。新功能如果會影響 demo 穩定性，先放到下一版。

### 2. README 缺少執行指令

不要只寫「執行程式」。請明確寫：

```powershell
python main.py
```

如果需要先安裝套件，也要寫清楚。

### 3. Demo 只講想法，沒有實際操作

成果發表一定要實際跑程式。可以用固定測試資料，避免現場輸入太多。

### 4. final_report 太空泛

不要只寫「我學到很多」。請寫具體例子，例如哪個錯誤訊息、哪段程式、哪個功能取捨。

## 修正方向

1. 先確認主流程能跑完。
2. 再補 README。
3. 再補測試資料。
4. 最後才調整畫面文字或加小功能。
5. Demo 前至少完整演練一次。

## 延伸練習

把專題整理成作品集格式：一張截圖、一段介紹、GitHub 連結、Demo 連結。
