# 第 15 週自我檢討：小型專題規劃與第一版原型

## 交作業前檢查

- [ ] `project_proposal.md` 有題目、使用者、MVP、功能拆分。
- [ ] 至少有一個 `.py` 檔可以執行。
- [ ] README 有執行方式與範例輸出。
- [ ] learning-log.md 有寫下縮小功能的取捨。

## 常見錯誤

### 1. 題目太大

「我要做完整記帳系統」太大。比較適合：

```text
可以新增花費、列出花費、計算總金額的記帳工具
```

### 2. 功能清單沒有分優先順序

請用：

- Must：一定要有，沒有就不能 demo
- Should：有會更完整，但沒有還能 demo
- Could：有時間再做

### 3. README 沒辦法讓別人跑起來

README 至少要寫：

```powershell
python main.py
```

以及一組範例輸入輸出。

### 4. 第一版原型只有空檔案

第 15 週至少要有一個真的能執行的功能，不要只交企劃。

## 修正方向

1. 把題目縮到兩週能完成。
2. 先完成最核心的資料輸入或資料讀取。
3. 不要一開始就做漂亮畫面。
4. 每完成一個功能，就更新 README。

## 延伸練習

替專題建立一份 `todo.md`，列出第 16 週每天要完成的事。
