# Python 高中生程式學習計畫

這是一份參考 `mentor-program-5th` 結構整理出的 Python 課程計畫，但對象改成高中生，節奏改為每週 3 到 6 小時。目標不是短期轉職，而是讓學生建立可持續的程式思維、資料處理能力與專題實作經驗。

## 這是什麼

這是一個 16 週加第 0 週的 Python 入門計畫。課程從電腦基礎、環境安裝、變數、條件、迴圈開始，逐步進到函式、資料結構、檔案、除錯，最後依學生程度選擇資料分析、API、物件導向與專題。

第 1 到第 10 週是核心必修，原則上不需要第三方套件。第 11 到第 14 週是進階應用週，適合有老師帶或已完成基礎作業的學生。若學生完全零基礎，建議把第 0 週拆成兩週，或先完成 `resources/computer-basics.md`。

這份 repo 的設計重點：

1. 每週有明確學習目標。
2. 每週都有能展示的小作品。
3. 作業包含程式題、簡答題與反思題。
4. 學生要先做，再看自我檢討。
5. 期末專題要求能讓別人照 README 跑起來。

## 適合對象

- 高中生，無程式經驗可。
- 會基本電腦操作，或願意先補檔案、資料夾、終端機等基礎。
- 願意每週固定練習 3 到 6 小時。
- 對資料分析、遊戲、網站、科展或自動化有興趣。

## 課程目標

### 核心能力

1. 能把問題拆成步驟，再翻譯成 Python 程式。
2. 能讀懂常見錯誤訊息，並定位出錯行數。
3. 能使用 list、dict、檔案與 CSV 處理資料。
4. 能用函式整理程式，避免所有邏輯擠在一起。
5. 能完成一個有 README、可執行、可展示的小作品。

### 工具能力

1. 電腦基礎：檔案、資料夾、副檔名、路徑、終端機目前位置。
2. Python：基礎語法、函式、資料結構、檔案、錯誤處理、模組。
3. 資料：CSV、pandas、matplotlib、簡單圖表解讀。
4. 網路：API、JSON、HTTP 基本概念。
5. 協作：Git、GitHub、README、學習日誌。

## 課程規則

### 1. 每週學習日誌

每週至少寫一篇學習日誌。建議回答：

- 本週完成了什麼？
- 最卡的是什麼？後來怎麼處理？
- 哪一段程式碼最值得保留？
- 下週要改進什麼？

### 2. 交作業前先自我檢查

交作業前請先確認：

- 程式可以從終端機執行。
- README 或註解有說明怎麼使用。
- 至少測過題目給的範例與一個邊界情況。
- 沒有把密碼、token、API key 放進 GitHub。

### 3. 簡答題不能只複製貼上

簡答題重點是理解。可以查資料，但請用自己的話整理，並附上參考來源連結。

### 4. 鼓勵問問題，但要帶著脈絡

提問時請附上：

1. 你想做到什麼。
2. 你目前寫了什麼。
3. 錯誤訊息完整內容。
4. 你已經試過哪些方法。

## 課程進行方式

建議每週節奏：

1. 讀本週 README 與指定教材。
2. 需要時先看 [`resources/videos.md`](resources/videos.md) 的補充影片。
3. 跟著範例打一遍，不只看。
4. 完成本週作業。
5. 看 `examples/weekN` 的自我檢討並修正。
6. 寫學習日誌。

## 評量方式

- 每週作業：50%
- 學習日誌：15%
- 課堂參與與互評：15%
- 期末專題：20%

評量不只看答案是否正確，也看命名、拆解、測試、README 與反思品質。

## 建議環境

第 1 到第 10 週只需要：

- Python 3.11 以上
- VS Code
- Git

第 11 週後才需要第三方套件：

- pandas
- matplotlib
- requests

安裝第三方套件：

```bash
pip install -r requirements.txt
```
