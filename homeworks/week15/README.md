# 第 15 週作業：小型專題規劃與第一版原型

## 本週目標

把前面學到的能力整合成一個小作品。這週不是要完成所有功能，而是要把題目縮小，做出「可以跑的第一版」。

## 本週主題

- 題目收斂
- 使用者是誰
- must / should / could 功能拆分
- 資料來源與資料格式
- 最小可行版本 MVP
- README 寫法
- 任務切分

## 專題選題原則

好的高中生專題不一定要大，但要清楚。請選一個你能在兩週內完成的題目。

建議題目：

- 讀書時間紀錄器
- 單字小測驗
- 班級問卷統計
- 個人記帳分類
- 社團活動報名整理
- 睡眠紀錄與圖表
- 文字冒險遊戲
- GitHub 使用者查詢工具

不建議題目：

- 完整社群網站
- 大型遊戲
- AI 聊天機器人
- 需要登入、付款、複雜資料庫的系統

### hw2：完成第一版核心功能

請至少完成一個可以執行的核心功能。例如：

- 記帳工具：可以新增一筆花費。
- 單字測驗：可以問 3 題並計分。
- 問卷統計：可以讀 CSV 並算人數。
- 文字遊戲：可以顯示第一個場景並讓玩家選擇。

### hw3：專題 README 第一版

建立 `README.md`，至少包含：

- 作品名稱
- 作品用途
- 如何執行
- 目前已完成功能
- 範例輸入與輸出
- 下一週要完成什麼

### hw4：簡答題

請寫在 `learning-log.md`：

1. 你為什麼選這個題目？
2. 你的 MVP 是什麼？
3. 哪個功能最容易超出時間？你打算如何縮小？
4. 如果只剩一天，你會保留哪一個功能？
