# 第 16 週作業：專題完成與成果發表

## 本週目標

完成一個可以展示、可以讓別人照 README 執行的 Python 小作品。這週重點是收尾、測試、說明與反思，不是臨時加一堆新功能。

## 本週主題

- 功能收尾
- 測試資料
- README 完整度
- Demo 流程
- 成果發表
- 同儕回饋
- 下一版規劃

## 必做作業

### hw1：完成專題

請完成第 15 週定義的 MVP。至少要符合：

- 程式可以從終端機執行。
- 核心功能可以正常使用。
- 有處理至少一種錯誤輸入。
- README 寫清楚如何執行。

### hw2：整理 README 最終版

README 至少包含：

- 專題名稱
- 作品介紹
- 使用方式
- 安裝需求
- 執行指令
- 範例輸入與輸出
- 已完成功能
- 未來可改進功能

### hw3：完成 final_report.md

建立 `final_report.md`，回答：

1. 我做了什麼作品？
2. 我原本想解決什麼問題？
3. 我完成了哪些功能？
4. 我遇到最困難的問題是什麼？
5. 我怎麼解決或繞過它？
6. 如果有下一版，我會改什麼？
7. 我最想展示哪一段程式碼？為什麼？

### hw4：Demo 準備

準備 3 到 5 分鐘 Demo，建議流程：

1. 30 秒：作品想解決什麼問題。
2. 2 分鐘：實際操作功能。
3. 1 分鐘：展示一段重要程式碼。
4. 30 秒：說明卡關與下一版。

### hw5：同儕回饋

看兩位同學作品，給出具體回饋。每份回饋至少包含：

- 一個做得清楚的地方
- 一個可以改善的地方
- 一個你看完學到的點

## 繳交內容

請繳交完整專題資料夾，至少包含：

- 主程式 `.py`
- 需要的 `data/` 或範例資料
- `README.md`
- `final_report.md`
- Demo 影片連結或簡報截圖
- `feedback.md`：給兩位同學的回饋

## 自我檢測

- [ ] 作品能從乾淨資料夾照 README 執行。
- [ ] README 有範例輸入與輸出。
- [ ] 我至少測過三種操作情境。
- [ ] 我能說明最重要的一段程式碼。
- [ ] 我能說明一個取捨：哪些功能做了，哪些功能先不做。
- [ ] final_report.md 有完整反思。

## 挑戰題

替專題加上 `sample_data/` 或 `demo_data/`，讓別人不用自己準備資料也能直接試跑。
